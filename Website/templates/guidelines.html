Guidelines for Successful Upload


To ensure a successful upload, please make sure ALL submissions comply with the following guidelines.
Files with a large number of rows may not upload. Files less than 1 MB should all upload properly, but files larger may currently cause the server to time out. We are working to expand the accepted file sizes largely.
Form submissions deviating from these guidelines will likely fail to be uploaded when processed:
Processing script URLs cannot contain the percent symbol (%)
If the processing script comes from Jupyter notebook, the best way to avoid this error is to rename the Jupyter Notebook file without any spaces – Spaces can be replaced with dashes or underscores instead



Files deviating from these guidelines will likely fail to be uploaded when processed:
All files must be tab-separated
No files can contain NA values – the submission will fail
Files must NOT include a header (i.e. no column names)
Files cannot have duplicate submission titles. If you receive an error regarding duplicate submission titles, please return to the ‘Contribute’ page and change your submission title



GMT Files should follow the following format for successful submission:
Standard GMT format
Column 1: Kinase/Protein/Transcription Factor of Interest
Naming of the FIRST term
No terms can be over 225 characters long
terms with multiple words as part of the name should NOT be separated by underscore or dash → If multiple words are part of the name, separate by spaces or by periods (.)
Next to the term, place an underscore (_) and write the species name of the term
if multiple words are part of the term name, please make sure this underscore with species is the LAST part written
Column 2: Description
No spaces in this column (must be a single word)
Can be left empty (although an additional tab would still be necessary between the gene set name and its terms), but for reliable performance use ‘None’ if no description
Columns 3→
All interaction proteins, each in their own column (variable number of columns per kinase/protein/transcription factor)
These terms are only the gene symbol (no indication of species needed as with the first term)
No terms can be over 225 characters long
Example GMT File
An example of a GMT file whose format is accepted and can be successfully submitted is found on the ‘HELP’ page of BioNetBay



SIG Files should follow the following format for successful submission:
Standard SIG format
All SIG files MUST have 13 columns corresponding to:
source name
source human accession
source mouse accession
source type
source location
target name
target human accession
target mouse accession
target type
target location
effect
type of interaction
PubMed IDs



Bolded columns (columns 1 and columns 6) MUST include source name and the target name
All other columns can simply be listed as NA
None of the names can be over 225 characters long
Example SIG file
An example of a SIG file whose format is accepted and can be successfully submitted is found on the ‘HELP’ page of BioNetBay
